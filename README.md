# Takiuddin Ahmed — Portfolio

Personal portfolio for **Md Takiuddin Ahmed** (Senior Backend Engineer), live at **[takiuddin.me](https://takiuddin.me)**.

It's a static single-page site served by a **Cloudflare Worker** running the **[Hono](https://hono.dev)** framework, with the static files delivered through **[Workers Static Assets](https://developers.cloudflare.com/workers/static-assets/)**.

---

## Architecture

```
Request ──▶ Cloudflare Worker (src/index.ts, Hono)
              1. security-headers middleware  (CSP, HSTS, … on every response)
              2. redirect routes              (/cv, /github, …)
              3. catch-all ──▶ ASSETS.fetch() ──▶ files in public/
                              + per-path Cache-Control / Content-Type / SEO headers
```

- **`src/index.ts`** — the Hono Worker. All header, redirect, caching, and 404 logic lives here (there are no `_headers` / `_redirects` files). `run_worker_first = true` means the Worker runs for **every** request, so security headers apply to static assets too.
- **`public/`** — the served asset directory. Everything reachable by URL is here; everything else (source, config, docs) stays at the repo root and is not served.
- **Caching / ETags** are handled automatically by Workers Static Assets (strong ETags + 304s). `Cache-Control` values are set per-path in `src/index.ts`.
- **`index.html` / `404.html` are fully self-contained** — all CSS and JS is inline. The only external resources are Google Fonts, Google Analytics, and the chat API (`ask-api.takiuddin.me`). Edit the inline `<style>` / `<script>` blocks to change the site.

## Quick start

**Prerequisites:** Node.js 18+ and npm.

```bash
npm install
npm run dev      # local Worker + assets at http://127.0.0.1:8787
```

### Commands

| Command | Description |
|---|---|
| `npm run dev` | `wrangler dev` — local Worker + static assets |
| `npm run deploy` | `wrangler deploy` — publish the Worker |
| `npm run types` | `wrangler types` — regenerate `worker-configuration.d.ts` (run after editing `wrangler.toml`) |
| `npm run typecheck` | `tsc --noEmit` |
| `npm run optimize:images` | Regenerate `public/assets/images/optimized/` (AVIF + WebP) via sharp |
| `npm run optimize:images:responsive` | Also emit `-sm/-md/-lg/-xl` responsive variants |

Wrangler bundles `src/index.ts` automatically — there is no separate build step for the pages.

## How requests are handled

All of this is implemented in `src/index.ts`:

- **Security headers** (CSP, HSTS, `X-Frame-Options`, `X-Content-Type-Options`, `Referrer-Policy`, `Permissions-Policy`) are applied to every response by a global middleware. To change an allowed origin, edit the `CSP` constant — keep it in sync with what the pages actually load or you'll break fonts / analytics / chat.
- **Redirects:** `/github`, `/linkedin`, `/calendly` (302 → external); `/cv`, `/resume` → resume PDF, `/image` → profile photo, `/favicon.ico` → favicon (301); `/gameapp.gov.bd` → `/` (301).
- **Caching:** `/assets/*` → 1-year immutable; `*.html`/`/` → 5 min fresh + `stale-while-revalidate` (instant load, background refresh); other top-level files → 1 hour; PDFs → 1 day.
- **SEO headers:** `X-Robots-Tag` on `robots.txt`, `llms*.txt`, and the resume PDF; `Access-Control-Allow-Origin: *` on the `llms*.txt` files.
- **404:** `not_found_handling = "404-page"` serves `public/404.html` with a 404 status for unmatched paths.

### SEO / AI surfaces

Keep these consistent with page content when facts change: the JSON-LD `@graph` in `public/index.html` `<head>`, `public/sitemap.xml`, `public/robots.txt`, `public/manifest.json`, and the hand-maintained `public/llms.txt` / `public/llms-full.txt`. `profile/taki_details.md` is the long-form source used to author them.

## Deployment

Deploys to Cloudflare Workers with Wrangler.

```bash
npm run deploy   # → takiuddinahmed-github-io.<subdomain>.workers.dev
```

**Custom domain cut-over (Pages → Workers):**

1. Verify the workers.dev deployment.
2. In the Cloudflare dashboard, **remove `takiuddin.me` from the old Pages project** (a domain can't be attached to both).
3. Uncomment the `routes` line in `wrangler.toml` and redeploy:
   ```toml
   routes = [{ pattern = "takiuddin.me", custom_domain = true }]
   ```
4. Zone-level rules (Always Use HTTPS, www→apex redirect) are configured in the Cloudflare dashboard and are unaffected by the migration.

Rollback: the Pages project is untouched and redeployable — re-attach the domain to it if needed.

## Troubleshooting

- **Types out of date after a `wrangler.toml` change** → `npm run types`, then `npm run typecheck`.
- **A resource is blocked in the browser console** → it's the CSP; add the origin to the `CSP` constant in `src/index.ts`.
- **`sharp` failed to install** → `npm install` again; it's an optional dependency used only by the image scripts.
